# Master Operating System

## Scopo

Questo documento descrive il sistema operativo interno di Opal Signal.

Opal Signal deve diventare una struttura capace di trasformare idee, posizionamento e obiettivi di acquisizione in sistemi digitali concreti.

## Principio centrale

Non costruire prima di aver chiarito cosa serve davvero.

Sequenza:

1. Ricerca.
2. Validazione.
3. Decisione.
4. Progettazione.
5. Costruzione.
6. Verifica.
7. Pubblicazione.

## Fase attuale

Fase 0: base tecnica e documentale.

## Regole operative

- MVP first.
- Funzione prima dell'estetica.
- Anti-overengineering.
- Documentare le decisioni importanti.
- Verificare con build e lint.
- Non creare strumenti complessi prima della base.

## Prossimo livello

Dopo la base:

- homepage reale;
- pubblicazione;
- standardizzazione;
- template cliente;
- component library;
- workflow piu' automatizzato.
